# 大学生竞赛活动管理平台 - 数据库设计说明

## 一、数据库概述

- **数据库名称**: competition
- **字符集**: utf8mb4
- **存储引擎**: InnoDB

## 二、表结构设计

### 1. 角色表 (role)
存储系统中的角色信息，包括学生、老师、管理员、评审员。

**主要字段**:
- `id`: 角色ID（主键）
- `name`: 角色名称
- `code`: 角色代码（用于程序识别）
- `description`: 角色描述

### 2. 用户表 (user)
统一管理所有用户信息（学生、老师、管理员、评审员）。

**主要字段**:
- `id`: 用户ID（主键）
- `username`: 用户名（登录账号，唯一）
- `password`: 密码（加密存储）
- `name`: 姓名
- `gender`: 性别
- `phone`: 手机号
- `email`: 邮箱
- `student_no`: 学号（学生专用）
- `college`: 学院
- `major`: 专业
- `grade`: 年级
- `avatar`: 头像URL
- `status`: 状态（启用/禁用）

### 3. 用户角色关联表 (user_role)
实现用户与角色的多对多关系，一个用户可以有多个角色。

**主要字段**:
- `user_id`: 用户ID（外键）
- `role_id`: 角色ID（外键）

### 4. 竞赛表 (competition)
存储竞赛的详细信息。

**主要字段**:
- `id`: 竞赛ID（主键）
- `title`: 竞赛标题
- `description`: 竞赛描述
- `type`: 竞赛类型
- `level`: 竞赛级别（校级、省级、国家级等）
- `organizer`: 主办单位
- `sponsor`: 承办单位
- `start_time`: 报名开始时间
- `end_time`: 报名结束时间
- `competition_time`: 竞赛时间
- `location`: 竞赛地点
- `max_participants`: 最大参赛人数
- `current_participants`: 当前报名人数
- `team_size`: 团队人数限制
- `status`: 竞赛状态
- `publisher_id`: 发布者ID（老师）
- `approver_id`: 审核者ID（管理员）
- `approval_status`: 审核状态

### 5. 报名表 (registration)
存储学生的报名信息。

**主要字段**:
- `id`: 报名ID（主键）
- `competition_id`: 竞赛ID（外键）
- `student_id`: 学生ID（外键）
- `team_name`: 团队名称（团队赛时使用）
- `team_leader_id`: 队长ID
- `contact_phone`: 联系电话
- `contact_email`: 联系邮箱
- `application_reason`: 申请理由
- `status`: 报名状态（待审核、已通过、已拒绝等）

### 6. 团队成员表 (team_member)
存储团队赛的成员信息。

**主要字段**:
- `registration_id`: 报名ID（外键）
- `student_id`: 学生ID（外键）
- `role`: 团队成员角色（队长、队员）

### 7. 作品表 (work)
存储学生提交的作品信息。

**主要字段**:
- `id`: 作品ID（主键）
- `registration_id`: 报名ID（外键）
- `competition_id`: 竞赛ID（外键）
- `student_id`: 提交者ID（外键）
- `title`: 作品标题
- `description`: 作品描述
- `file_url`: 作品文件URL
- `file_name`: 作品文件名
- `file_size`: 文件大小
- `submit_time`: 提交时间
- `status`: 作品状态（待评审、评审中、已评审等）

### 8. 评分表 (score)
存储评审员对作品的评分信息。

**主要字段**:
- `id`: 评分ID（主键）
- `work_id`: 作品ID（外键）
- `reviewer_id`: 评审员ID（外键）
- `score`: 评分（0-100分）
- `score_detail`: 评分详情（JSON格式，可存储多个评分项）
- `comment`: 评审意见
- `review_time`: 评审时间

### 9. 获奖记录表 (award)
存储获奖信息。

**主要字段**:
- `id`: 获奖ID（主键）
- `competition_id`: 竞赛ID（外键）
- `registration_id`: 报名ID（外键）
- `work_id`: 作品ID（外键）
- `student_id`: 获奖学生ID（外键）
- `award_level`: 获奖等级（一等奖、二等奖等）
- `award_rank`: 获奖排名
- `certificate_url`: 证书URL
- `prize`: 奖品
- `total_score`: 总得分

### 10. 操作日志表 (operate_log)
记录系统操作日志（可选）。

**主要字段**:
- `id`: 日志ID（主键）
- `user_id`: 操作用户ID（外键）
- `operation`: 操作类型
- `method`: 操作方法
- `params`: 请求参数
- `result`: 返回结果
- `ip`: IP地址
- `create_time`: 创建时间

## 三、功能实现说明

### 学生功能
1. **申请报名**: 通过 `registration` 表记录报名信息
2. **提交作品**: 通过 `work` 表记录作品信息
3. **查询过往报名记录**: 查询 `registration` 表，关联 `competition` 表
4. **查询获奖成绩**: 查询 `award` 表，关联 `competition` 表

### 老师功能
1. **发布竞赛信息**: 在 `competition` 表中插入记录，`publisher_id` 为老师ID，`approval_status` 为待审核

### 管理员功能
1. **统计学生人数**: 查询 `user` 表，关联 `user_role` 表，筛选角色为学生的用户
2. **管理老师发布竞赛**: 通过 `competition` 表的 `approval_status` 字段进行审核，`approver_id` 为管理员ID

### 评审员功能
1. **对学生提交作品打分**: 在 `score` 表中插入评分记录，`reviewer_id` 为评审员ID

## 四、表关系图

```
user (用户)
  ├── user_role (用户角色关联) ──> role (角色)
  ├── competition (竞赛) [publisher_id, approver_id]
  ├── registration (报名) [student_id, team_leader_id]
  ├── team_member (团队成员) [student_id]
  ├── work (作品) [student_id]
  ├── score (评分) [reviewer_id]
  ├── award (获奖) [student_id]
  └── operate_log (操作日志) [user_id]

competition (竞赛)
  ├── registration (报名) [competition_id]
  ├── work (作品) [competition_id]
  └── award (获奖) [competition_id]

registration (报名)
  ├── team_member (团队成员) [registration_id]
  ├── work (作品) [registration_id]
  └── award (获奖) [registration_id]

work (作品)
  ├── score (评分) [work_id]
  └── award (获奖) [work_id]
```

## 五、使用说明

1. 执行 `db_schema.sql` 文件创建所有表结构
2. 系统会自动初始化角色数据和管理员账号
3. 默认管理员账号：
   - 用户名: admin
   - 密码: admin123（实际使用时需要加密存储）

## 六、注意事项

1. **密码加密**: 实际使用时，用户密码必须使用BCrypt等加密算法加密存储
2. **文件存储**: 作品文件建议使用OSS等对象存储服务，`file_url` 存储文件访问地址
3. **评分详情**: `score_detail` 字段使用JSON格式存储，可以灵活存储多个评分项
4. **索引优化**: 已为常用查询字段创建索引，提升查询性能
5. **外键约束**: 使用外键保证数据完整性，删除时使用CASCADE或SET NULL策略
